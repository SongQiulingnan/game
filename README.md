# 挑夹棋 - Vue 3 + TypeScript 重构版

这是一个使用 Vue 3 + TypeScript + Vite 重构的挑夹棋游戏。

## 项目结构

```
frontend/
├── src/
│   ├── components/          # Vue组件
│   │   ├── Board.vue        # 棋盘组件
│   │   ├── Controls.vue     # 控制按钮组件
│   │   ├── InfoPanel.vue    # 信息面板组件
│   │   ├── RulesBox.vue     # 规则说明组件
│   │   └── StatusBar.vue    # 状态栏组件
│   ├── composables/         # 组合式函数
│   │   ├── useGame.ts       # 游戏逻辑
│   │   └── useAI.ts         # AI逻辑
│   ├── types/               # TypeScript类型定义
│   │   └── index.ts
│   ├── utils/               # 工具函数
│   │   ├── board.ts         # 棋盘工具函数
│   │   ├── boardLines.ts    # 棋盘连线定义
│   │   └── constants.ts     # 常量定义
│   ├── App.vue              # 主应用组件
│   ├── main.ts              # 入口文件
│   └── style.css            # 全局样式
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## 功能特性

- ✅ 完整的挑夹棋游戏逻辑
- ✅ 支持单步/多步移动模式

## 游戏规则

### 一、棋盘与初始布局

- **主棋盘**：4×4 方格，共 5×5 个交点，包含两条对角线
- **顶部菱形区域**（田字/山）：位于主棋盘正上方，内部有十字交叉线，共 4 个交点
- **初始棋子**：
  - 黑方 5 枚棋子，初始位于主棋盘左侧一列（x=0）
  - 白方 5 枚棋子，初始位于主棋盘右侧一列（x=4）
  - 双方各有 5 枚备用棋子，用于吃子后补位

### 二、移动规则（双模式）

游戏支持一步模式与多步模式，可在对局中随时切换。

#### 2.1 一步模式
- 每步只能沿棋盘连线移动一格
- 落点必须是相邻的空交点

#### 2.2 多步模式
- 棋子可沿直线（横、竖、对角）移动任意格数
- 不可越过其他棋子，遇到棋子即停止该方向移动
- 落点必须为空交点

> 注：移动仅可在网格交点上进行，棋子必须沿现有连线移动。

### 三、吃子规则

吃子分为夹吃与挑吃两种，黑白双方共用。

#### 3.1 夹吃
- **条件**：在同一直线上，两枚己方棋子紧夹一枚敌方棋子（三子紧密相邻，形如 己—敌—己）
- **限制**：该吃子结构两端紧邻的延长点上不能有**任何棋子**，且仅限此三子都在的那一条线上
- **执行**：吃掉被夹的敌方棋子，并在其原位补上一枚己方备用棋子

#### 3.2 挑吃
- **条件**：一枚己方棋子移动到两枚敌方棋子中间（三子紧密相邻，形如 敌—己—敌）
- **限制**：该吃子结构两端紧邻的延长点上不能有**任何棋子**，且仅限此三子都在的那一条线上
- **执行**：同时吃掉两枚敌方棋子，并在两枚被吃棋子的原位各补上一枚己方备用棋子

#### 3.3 吃子失败条件
- 若吃子结构两端紧邻的延长点上存在**任何棋子**（无论己方或敌方），仅限此三子都在的那一条线上紧邻的两个点位，则此次吃子失败，不执行吃子
- 若备用棋子数量不足，无法补位，则不能吃子（此情况极少见）

### 四、连环吃子（扩展规则）

- 吃子后，如果补位的棋子又形成了新的吃子条件（夹吃或挑吃），则继续执行吃子
- 连环吃子会自动进行，无需玩家手动触发
- 连环吃子无次数限制，直到不能继续吃子为止
- 补位的棋子可以参与后续吃子

### 五、弱势保护（≤ 2 子不可吃）

- 当敌方棋盘上棋子数量 **≤ 2** 枚时，该方棋子**完全不可被吃**（夹吃和挑吃均不生效）
- 此规则保护复活后的弱势方：复活后只有 2 子，无法被吃
- 若复活落子时恰好吃掉对方棋子使得对方变为 ≤ 2 子，对方同样受保护
- 只有敌方棋子数量 **> 2** 时，夹吃和挑吃才正常执行
- 此时只能通过围困使敌方无路可走，从而获胜

### 六、胜负规则

#### 6.1 围困获胜
- 将敌方棋子吃到仅剩一枚
- 将该枚棋子围困在顶部菱形区域内，使其无法移动
- 同时满足以上两点，即判获胜

#### 6.2 无子可动获胜
- 若敌方棋子数量大于 1，但全局无任何棋子可以移动，则当前行棋方直接获胜

### 七、复活机制

当同时满足以下三个条件时，触发复活：
1. 己方棋盘上仅剩最后一枚棋子
2. 该棋子不在顶部菱形区域内（即在主棋盘区域）
3. 该棋子当前无法移动（根据当前移动模式判定无路可走）

**复活操作**：
- 触发后，玩家可在主棋盘（菱形区域外）任意空交点落子
- 落子需消耗 1 枚己方备用棋子
- 复活后，己方棋子恢复为 2 枚，回合自动交给对方
- 若备用棋子已耗尽，则无法触发复活

### 八、其他说明

- 所有棋子只能在网格交点上移动，不可脱离连线
- 吃子后补位使用的棋子来自备用棋子池，棋盘上的棋子总数可能因吃子与补位而动态变化
- 游戏支持随时重新开始，并可在对局中自由切换一步/多步模式

## 技术栈

- **前端框架**: Vue 3 + Composition API
- **类型系统**: TypeScript
- **构建工具**: Vite
- **代码规范**: ESLint + Prettier

## 项目特点

1. **组件化架构**: 游戏被拆分为多个独立的Vue组件，每个组件职责单一
2. **组合式函数**: 使用Vue 3的Composition API，将游戏逻辑封装在composables中
3. **类型安全**: 全面使用TypeScript，提供完整的类型定义
4. **响应式设计**: 游戏界面适配不同屏幕尺寸，支持移动端和PC端
5. **模块化**: 工具函数、类型定义、常量等分离到不同模块

## 开发说明

### 添加新功能
1. 在`types/index.ts`中定义新的类型
2. 在`utils/`中添加工具函数
3. 在`composables/`中添加业务逻辑
4. 在`components/`中创建新的Vue组件
5. 在`App.vue`中集成新组件

### 代码规范
- 使用ESLint进行代码检查
- 使用Prettier进行代码格式化
- 遵循Vue 3的Composition API最佳实践
